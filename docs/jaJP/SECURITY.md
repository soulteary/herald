このドキュメントは、Herald のセキュリティ機能、セキュリティ設定、ベストプラクティスについて説明します。

> ⚠️ **注意**: このドキュメントは翻訳中です。完全なバージョンについては、[英語版](../enUS/SECURITY.md)を参照してください。

## 実装されたセキュリティ機能

1. **チャレンジベースの検証**: リプレイ攻撃を防ぎ、検証コードの一回限りの使用を保証するために challenge-verify モデルを使用
2. **安全なコードストレージ**: 検証コードは Argon2 ハッシュとして保存され、プレーンテキストでは保存されません
3. **多次元レート制限**: user_id、宛先（メール/電話）、IP アドレスによるレート制限で悪用を防止
4. **サービス認証**: サービス間通信のために mTLS、HMAC 署名、API キー認証をサポート
5. **べき等性保護**: べき等キーを使用してチャレンジの重複作成とコードの重複送信を防止
6. **チャレンジの有効期限**: 設定可能な TTL でチャレンジの自動有効期限切れ
7. **試行回数制限**: ブルートフォース攻撃を防ぐためのチャレンジごとの最大試行回数制限
8. **再送信クールダウン**: 検証コードの迅速な再送信を防止
9. **監査ログ**: 送信、検証、失敗を含むすべての操作の完全な監査証跡
10. **プロバイダーセキュリティ**: メールおよび SMS プロバイダーとの安全な通信

詳細については、[英語版](../enUS/SECURITY.md)を参照してください。

## 脆弱性の報告

セキュリティの脆弱性を発見した場合は、以下を通じて報告してください：

1. **GitHub Security Advisory**（推奨）
   - リポジトリの [Security タブ](https://github.com/soulteary/herald/security) に移動
   - "Report a vulnerability" をクリック
   - セキュリティアドバイザリフォームに記入

2. **メール**（GitHub Security Advisory が利用できない場合）
   - プロジェクトのメンテナーにメールを送信
   - 脆弱性の詳細な説明を含める

**公開の GitHub Issues を通じてセキュリティの脆弱性を報告しないでください。**
